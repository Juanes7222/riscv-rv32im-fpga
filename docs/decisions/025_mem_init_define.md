# ADR 025 — Memory Initialization: `.mem` Path Injection via `\`define` Preprocessor Macro

**Status:** Accepted
**Date:** 2026-05-04
**Supersedes:** `parameter string MEM_FILE` approach attempted during early RTL development.
**Depends on:** ADR 013 (memory depths), ADR 014 (`.mem` format and generation pipeline)

## Context

Both `instruction_memory.sv` and `data_memory.sv` require a mechanism to inject
the path of their initialization files into `$readmemh` at elaboration time.
The `.mem` file format and the ELF-to-.mem conversion pipeline are fully defined
in ADR 014 and are not reconsidered here. This ADR exclusively addresses how those
`.mem` file paths are communicated to the RTL without hardcoding them in the source.



## Decision

Memory file paths are specified via preprocessor macros
defined in a generated header file `mem_config.vh`. Both `instruction_memory.sv`
and `data_memory.sv` `` `include `` this header and use the macro directly in
`$readmemh`.

This ADR only specifies how `.mem` file paths are passed into the RTL. It does not
change the `.mem` file format or the ELF-to-`.mem` conversion pipeline, which are
fully defined in ADR 014. Any previous use of `objcopy -O verilog` for `.mem`
generation is superseded by ADR 014.

The header file is generated by a Python script before synthesis or simulation and
is **not committed to version control** (listed in `.gitignore`). A template with
placeholder paths is committed as `mem_config.vh.template`.

## Normative Specification

### File: `rtl/shared/mem_config.vh` (generated, not committed)

```verilog
// Generated by scripts/gen_mem_config.py — do not edit manually
`ifndef MEM_CONFIG_VH
`define MEM_CONFIG_VH

`define IMEM_FILE "path/to/program.mem"
`define DMEM_FILE "path/to/data.mem"

`endif
```

### File: `rtl/shared/mem_config.vh.template` (committed)

```verilog
// Copy to mem_config.vh and fill in paths before synthesis or simulation
`ifndef MEM_CONFIG_VH
`define MEM_CONFIG_VH

`define IMEM_FILE "FILL_IN_IMEM_PATH"
`define DMEM_FILE "FILL_IN_DMEM_PATH"  // Leave empty string if no data init

`endif
```

### Usage in `instruction_memory.sv`

```systemverilog
`include "mem_config.vh"

module instruction_memory #(
    parameter integer IMEM_DEPTH = 16384  // 64 KB (ADR 013)
) (
    input  logic [31:0] addr,
    output logic [31:0] instruction
);
    logic [31:0] mem [0:IMEM_DEPTH-1];

    initial begin
        $readmemh(`IMEM_FILE, mem);
    end

    assign instruction = mem[addr[31:2]];  // word-addressed, ignore addr[1:0]

endmodule
```

### Usage in `data_memory.sv`

```systemverilog
`include "mem_config.vh"

module data_memory #(
    parameter integer DMEM_DEPTH = 8192   // 32 KB (ADR 013)
) (
    input  logic        clk,
    input  logic [31:0] addr,
    input  logic [31:0] wr_data,
    input  logic        dm_wr,
    input  logic [2:0]  dm_ctrl,
    output logic [31:0] rd_data
);
    logic [7:0] mem [0:DMEM_DEPTH*4-1];  // byte-organized (ADR 021)

    initial begin
        `ifdef DMEM_FILE
            if (`DMEM_FILE != "")
                $readmemh(`DMEM_FILE, mem);
        `endif
    end

    // ... byte-lane read/write logic (ADR 019, ADR 020, ADR 021)

endmodule
```

## Generator Script: `scripts/gen_mem_config.py`

```python
#!/usr/bin/env python3
"""
Generate rtl/shared/mem_config.vh from command-line paths.
Usage:
    python scripts/gen_mem_config.py \
        --imem path/to/program.mem \
        --dmem path/to/data.mem      # optional
"""
import argparse, pathlib

TEMPLATE = """// Generated by scripts/gen_mem_config.py — do not edit manually
`ifndef MEM_CONFIG_VH
`define MEM_CONFIG_VH

`define IMEM_FILE "{imem}"
`define DMEM_FILE "{dmem}"

`endif
"""

parser = argparse.ArgumentParser()
parser.add_argument("--imem", required=True)
parser.add_argument("--dmem", default="")
args = parser.parse_args()

out = pathlib.Path("rtl/shared/mem_config.vh")
out.write_text(TEMPLATE.format(imem=args.imem, dmem=args.dmem))
print(f"Written: {out}")
```

## Quartus Integration

In the Quartus project, `mem_config.vh` must be on the Verilog include path.
Add the following line to the `.qsf` file:

```tcl
set_global_assignment -name SEARCH_PATH rtl/shared
```

Alternatively, add the file explicitly:

```tcl
set_global_assignment -name VERILOG_INCLUDE_FILE rtl/shared/mem_config.vh
```

This ensures Quartus resolves the `` `include `` directive during synthesis.

## Icarus Verilog / cocotb Integration

When running cocotb with Icarus Verilog, pass the include directory via the
Makefile:

```makefile
COMPILE_ARGS += -I$(PWD)/rtl/shared
```

Or equivalently with the cocotb runner API:

```python
runner.build(
    includes=["rtl/shared"],
    ...
)
```

The same `mem_config.vh` used for synthesis is used for simulation, guaranteeing
that synthesis and simulation load identical program binaries. This eliminates the
dual-artifact risk identified in ADR 014.

## `.mem` File Format

The `.mem` files consumed by `$readmemh` conform to the format defined in ADR 014:
one 32-bit word per line in hexadecimal (no `0x` prefix, no address markers),
little-endian byte order, padded to the full memory depth with `00002013`
(NOP: `ADDI x0, x0, 0`) when the program binary is shorter than `DEPTH` words.

`.mem` files are generated from ELF binaries via the pipeline defined in ADR 014:
ELF → (objcopy -O binary) → .bin → (scripts/elf_to_mem.py) → .mem


This ADR does not redefine that pipeline. The `gen_mem_config.py` script
defined here only configures where those already-generated `.mem` files are located.

## Consequences

- `mem_config.vh` is added to `.gitignore`. `mem_config.vh.template` is committed.
- Changing the benchmark program requires only regenerating `mem_config.vh` via
  `gen_mem_config.py`, no RTL file is modified.
- Synthesis replicas (ADR 024, Anteproyecto §7.1.3) use the same `mem_config.vh`
  for all five runs of a given benchmark, guaranteeing identical program content
  across replicas.
- Both Icarus Verilog (via cocotb) and Quartus use the same include mechanism,
  eliminating the risk of loading different binaries in simulation and synthesis.
- `data_memory.sv` uses a conditional `$readmemh`: if `DMEM_FILE` is empty,
  the memory initializes to zero (Quartus and Icarus both zero-initialize
  uninitialized `logic` arrays).
- `parameter integer IMEM_DEPTH` and `DMEM_DEPTH` remain proper integer parameters
  and are not affected by this mechanism.
- This ADR is orthogonal to ADR 014. All scripts and Makefiles that produce `.mem`
  artifacts must follow ADR 014's pipeline; this ADR only governs how those
  artifacts are wired into the RTL.
